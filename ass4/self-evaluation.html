Introduction

A web pages accessibility is more than a checklist of technical specifications the developer must follow. It is a way to ensure that users will not be confused when entering your page. It is a way to ensure users comfort. Without them it is very possible that users will simply leave the page without considering the information provided within.
But we also need to consider that no web page can be perfect. There are times where developers must sacrifice certain aspects to better the end result. Knowing the target market will make the deciding job easier.

Being aware that we sell phones in our web page, we are aware of our target market. Such a website should be easily navigable, to ensure users comfort. we need to create a trusting base with our user, so that they'd want to buy from us again. We also need to approach with a trustworthy interface. To make sure that we achieved that we checked with the guidelines provided by the WCAG.

Accessibility

The most important guidelines for our web page are: 1.1, 1.3 and 2.4

1.1 Provide text alternatives:

  • Our page must be percievable by everyone. As a web shop it is very important that everyone is able to percieve every element of the page, including images, so that they have the best experience possible buying phones from us. If they cannot see the media, there should always be a tag describing what exactly it is that they should be seeing. Our web shop relies on pictures of the phones, so the least a user should expect is some alternative to those pictures if they cannot see them.

1.3 Content can be presented in different ways:

  • As it is a web shop, the page should not soley rely on layout either. It needs to be a simple page that is usable with or without fancy CSS layout structures. The importance of our page is what we sell, not the way we portray said products. Therefore the focus should be a clear structured page, rather than an abnoxiously decorated one that steals the focus away from our information.

2.4 Easy to Navigate:

  • Our page must be easily operable by everyone. This means that it should be fully navigable. A user should always know where they are in our page and how to get where they want to go. The user should always have a good experience and not be confused by our pages structure. A bad navigability hinders the users want to buy from us. Insted on focusing on buying a good phone, they would be to confused and would probably choose a different site to buy from.

Improvement Plan

For a better accessibility and usability of our web page we decided to implement the below listed improvements. They are listed in order of priority:

  1. To improve user experience for mobile users -> Improving our responsive design, eliminating all horizontal scrolling (making elements such as images smaller to fit into the screen)
  2. To improve the page usability when printing -> Defining the print-style in the CSS-file (adding media type print and adding widows/orphans)
  3. To improve page usability -> Passing both validator checks, fixing possible errors that we may have overlooked that made the page worse
  4. To have a neat and clear structured HTML-file -> Eliminating all CSS oriented markup from our HTML-file (br tags, width and height tags from our images)
  5. To have a complete page -> Implementing the missing items from the appendix (background, border-style, border-width, list-style-image, margin-bottom, orphans, widows)
  6. To fulfill the assignments needs -> adding source code for ass2 js code
